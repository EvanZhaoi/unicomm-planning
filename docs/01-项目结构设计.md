# UniComm 项目结构设计

## 一、项目拆分策略

### 1.1 仓库拆分原则

| 仓库 | 类型 | 说明 |
|------|------|------|
| unicomm-planning | 文档仓库 | 项目规划、需求文档、技术方案（当前） |
| unicomm-server | 后端服务 | Spring Boot 后端 API |
| unicomm-web | 前端应用 | React 单页应用 |
| unicomm-plugin | 插件系统 | 未来：可嵌入第三方系统的 SDK |

## 二、Server 项目结构（unicomm-server）

### 2.1 模块划分

```
src/main/java/com/unicomm/
├── unicomm-starter/          # 启动模块（不含业务代码）
│   ├── UniCommApplication.java
│   └── config/
│
├── module-common/           # 公共模块
│   ├── result/              # 统一返回结构
│   ├── exception/           # 全局异常处理
│   ├── validation/           # 参数校验
│   └── constant/            # 常量定义
│
├── module-memo/             # Memo 模块（当前唯一模块）
│   ├── controller/
│   ├── service/
│   ├── mapper/
│   ├── entity/
│   ├── dto/
│   └── vo/
│
├── module-im/               # IM 模块（未来扩展）
├── module-notify/           # Notify 模块（未来扩展）
├── module-ai/               # AI 模块（未来扩展）
├── module-plugin/           # 插件系统（未来扩展）
│
└── framework/               # 框架层
    ├── security/            # 安全认证
    ├── websocket/           # WebSocket 处理
    └── plugin/              # 插件机制
```

### 2.2 目录规范

| 目录 | 职责 | 规范 |
|------|------|------|
| controller | 接收请求、参数校验、调用 service | 禁止写业务逻辑 |
| service | 业务逻辑处理、事务管理 | 禁止直接操作数据库 |
| mapper | 数据库操作（MyBatis Plus） | 禁止写业务逻辑 |
| entity | 数据库实体，与表一一对应 | 禁止添加业务字段 |
| dto | 数据传输对象，请求参数 | 用于接收前端数据 |
| vo | 视图对象，响应数据 | 用于返回给前端 |
| config | 配置类 | 按功能划分 |

### 2.3 包边界规范

```
❌ 禁止
com.unicomm.controller.memo.MemoController
com.unicomm.service.MemoService
com.unicomm.mapper.MemoMapper

✅ 正确（按模块分包）
com.unicomm.module.memo.controller.MemoController
com.unicomm.module.memo.service.MemoService
com.unicomm.module.memo.mapper.MemoMapper
```

### 3.2 组件分层

```
┌─────────────────────────────────────┐
│          Pages（页面层）             │  只做：布局、状态展示、路由跳转
├─────────────────────────────────────┤
│        Features（功能层）             │  只做：业务逻辑、数据处理
├─────────────────────────────────────┤
│       Components（组件层）            │  只做：UI 展示、可复用
├─────────────────────────────────────┤
│           UI（基础层）               │  只做：最底层组件、不含业务逻辑
└─────────────────────────────────────┘
```

### 3.3 状态管理策略

| 场景 | 方案 | 说明 |
|------|------|------|
| 服务端数据（列表、详情） | TanStack Query | 缓存、同步、加载状态 |
| 客户端临时状态 | React useState | 不需要全局共享 |
| 跨组件状态 | Zustand | 用户信息、UI 状态 |
| 表单状态 | React Hook Form | 复杂表单校验 |

### 3.4 组件抽离规范

**当前端需要抽离到 plugin 时：**

```
unicomm-web/src/components/
├── ui/                    # 可抽离到 plugin
│   ├── Button.tsx
│   ├── Input.tsx
│   └── Modal.tsx
└── business/              # 不可抽离
    ├── MemoEditor.tsx
    └── MemoList.tsx
```

## 四、数据库设计原则

### 4.1 表命名规范

```
前缀：uni_

示例：
- uni_memo          # Memo 表
- uni_memo_group    # Memo 分组表
- uni_memo_tag      # Memo 标签表
- uni_memo_tag_rel  # Memo-标签关联表
```

### 4.2 公共字段

所有表必须包含：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键，自增 |
| create_time | datetime | 创建时间 |
| update_time | datetime | 更新时间 |
| create_user | bigint | 创建人 |
| deleted | tinyint | 逻辑删除（0=未删，1=已删） |

### 4.3 索引规范

- 主键索引：id
- 业务索引：name、status、create_time 等高频查询字段
- 外键索引：create_user、group_id 等关联字段

## 五、API 设计原则

### 5.1 统一返回格式

```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

| code | 说明 |
|------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未登录 |
| 403 | 无权限 |
| 500 | 服务器错误 |

### 5.2 RESTful 规范

| 操作 | Method | Path | 说明 |
|------|--------|------|------|
| 列表 | GET | /memos | 分页查询 |
| 详情 | GET | /memos/{id} | 获取单个 |
| 创建 | POST | /memos | 创建 |
| 更新 | PUT | /memos/{id} | 更新 |
| 删除 | DELETE | /memos/{id} | 删除（逻辑删除） |

### 5.3 分页查询

```
GET /memos?page=1&size=20&keyword=xxx&groupId=1&status=normal
```

响应：
```json
{
  "code": 200,
  "data": {
    "list": [...],
    "total": 100,
    "page": 1,
    "size": 20
  }
}
```
