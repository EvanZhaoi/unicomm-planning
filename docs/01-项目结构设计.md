# UniComm 项目结构设计

## 一、项目定位

**UniComm 是企业级桌面协作应用，不是普通 Web 管理后台。**

核心特点：
- 长期运行的后台应用
- 系统托盘常驻
- WebSocket 保持连接
- 支持离线工作
- 本地数据缓存
- 桌面原生能力

---

### 2.1 仓库拆分原则

| 仓库 | 类型 | 说明 |

## 四、Server 项目结构（unicomm-server）

### 4.1 模块划分

```
src/main/java/com/unicomm/
├── unicomm-starter/          # 启动模块（不含业务代码）
│   ├── UniCommApplication.java
│   └── config/
│
├── module-common/           # 公共模块
│   ├── result/              # 统一返回结构
│   ├── exception/           # 全局异常处理
│   ├── validation/           # 参数校验
│   └── constant/             # 常量定义
│
├── module-memo/             # Memo 模块（当前唯一模块）
│   ├── controller/
│   ├── service/
│   ├── mapper/
│   ├── entity/
│   ├── dto/
│   └── vo/
│
├── module-im/               # IM 模块（未来扩展）
├── module-notify/           # Notify 模块（未来扩展）
├── module-ai/               # AI 模块（未来扩展）
├── module-plugin/           # 插件系统（未来扩展）
│
└── framework/               # 框架层
    ├── security/            # 安全认证
    ├── websocket/            # WebSocket 处理
    └── plugin/               # 插件机制
```

### 4.2 目录规范

| 目录 | 职责 | 规范 |
|------|------|------|
| controller | 接收请求、参数校验、调用 service | 禁止写业务逻辑 |
| service | 业务逻辑处理、事务管理 | 禁止直接操作数据库 |
| mapper | 数据库操作（MyBatis Plus） | 禁止写业务逻辑 |
| entity | 数据库实体，与表一一对应 | 禁止添加业务字段 |
| dto | 数据传输对象，请求参数 | 用于接收前端数据 |
| vo | 视图对象，响应数据 | 用于返回给前端 |
| config | 配置类 | 按功能划分 |

## 五、桌面端能力清单

### 5.1 基础能力

| 能力 | 说明 | 优先级 |
|------|------|--------|
| 系统托盘 | 最小化到托盘、托盘菜单、托盘通知 | P0 |
| 多窗口 | 主窗口、设置窗口、通知窗口 | P1 |
| 窗口管理 | 置顶、最小化、最大化、关闭 | P0 |
| 自动更新 | 检测更新、下载更新、提示重启 | P2 |
| 全局快捷键 | 自定义快捷键触发功能 | P1 |

### 5.2 通信能力

| 能力 | 说明 | 优先级 |
|------|------|--------|
| WebSocket | 与后端保持长连接、消息推送 | P0 |
| 重连机制 | 网络断开自动重连 | P0 |
| 心跳检测 | 检测连接状态 | P0 |

### 5.3 本地能力

| 能力 | 说明 | 优先级 |
|------|------|--------|
| 本地缓存 | 热点数据本地缓存、离线支持 | P0 |
| 本地数据库 | SQLite 预留（未来离线扩展） | P2 |
| 系统通知 | 桌面通知、静默通知 | P1 |
| 剪贴板 | 读取/写入剪贴板 | P1 |
| 文件拖拽 | 拖拽文件到应用 | P2 |
| 本地文件系统 | 读取/保存本地文件 | P2 |

### 5.4 离线能力

| 能力 | 说明 | 优先级 |
|------|------|--------|
| 离线检测 | 检测网络状态 | P0 |
| 离线操作 | 离线时操作入队、联网后同步 | P1 |
| 冲突处理 | 离线编辑与在线数据冲突解决 | P2 |
