## 二、目录职责说明

### 2.1 按领域拆分 vs 按层级拆分

| 目录 | 说明 | 原则 |
|------|------|------|
| `stores/*` | 全局状态，按领域拆分 | 不要做页面级 store |
| `features/*` | 功能模块，按领域拆分 | 包含 API、Store、Hooks、组件 |
| `desktop/*` | 桌面能力，与 Tauri 交互 | 封装原生能力，供 features 调用 |
| `native/*` | 原生 API 抽象 | 提供统一的调用接口 |

### 2.1.1 当前实现说明

当前 MVP 为了降低拆分成本，托盘、后台运行、快捷键注册、快速 Memo 窗口创建先集中在 `src-tauri/src/lib.rs` 中实现。后续进入结构整理阶段时，再按上面的规划拆分到 `commands/*` 与 `services/*`。

已实现的桌面能力：

| 能力 | 说明 |
|------|------|
| Windows 用户识别 | 读取 username，并从 `USERDOMAIN / USERDNSDOMAIN` 获取企业域；本机账户不把计算机名当作 domain |
| Tauri 字段映射 | Rust 命令返回 `camelCase`，与前端 `deviceId/computerName/osVersion` 类型保持一致 |
| 系统托盘 | 使用 Tauri 2 tray API 创建托盘图标和显示/隐藏/退出菜单 |
| 后台运行 | 主窗口关闭时隐藏到后台，不退出应用 |
| 自定义标题栏 | 主窗口关闭原生 decorations，由 React 标题栏负责拖拽、最小化、最大化、关闭，背景跟随主题 |
| 主界面快捷键 | 默认 `Ctrl + Alt + M`，显示并聚焦主窗口 |
| 快速 Memo 快捷键 | 默认 `Ctrl + Alt + N`，显示快速 Memo 小窗口 |
| 快捷键设置 | 设置页修改后，通过 Tauri 命令动态重新注册 |
| 快速 Memo 窗口 | 独立隐藏窗口，仅用于新增 Memo |
| 语言设置 | 首次启动根据系统语言选择默认值；设置页支持中文/日文切换，持久化到本地 |
| 字体设置 | 全局使用内置阿里巴巴普惠体 3.0 / Alibaba Sans JP 字体文件 |
| Memo 工作台 | 支持列表、搜索、分组筛选、状态筛选、标题/内容编辑、Milkdown Crepe 可视化编辑、MD 源码切换、收藏、归档、保存 |

### 2.2 components/ vs features/

| 目录 | 说明 | 可抽离到 Plugin |
|------|------|----------------|
| components/ui | 基础 UI 组件 | ✅ 是 |
| components/layout | 布局组件 | ✅ 部分 |
| components/business | 业务组件 | ⚠️ 视情况 |
| features/memo | 功能模块 | ❌ 否 |

## 三、状态管理规范

### 3.1 状态管理选择

| 场景 | 方案 | 说明 |
|------|------|------|
| 服务端数据 | TanStack Query | 缓存、同步、离线支持 |
| 客户端临时状态 | React useState | 不需要全局共享 |
| 全局共享状态 | Zustand | 按领域拆分 store |
| 表单状态 | React Hook Form | 复杂表单校验 |

### 3.2 Zustand Store 规范

**按领域拆分的 Store：**

```typescript
// stores/memo.store.ts
export const useMemoStore = create<MemoState>()((set, get) => ({
  memos: [],
  currentMemo: null,
  setCurrentMemo: (memo) => set({ currentMemo: memo }),
  // ...
}));

// stores/socket.store.ts
export const useSocketStore = create<SocketState>()((set) => ({
  connected: false,
  reconnecting: false,
  setConnected: (connected) => set({ connected }),
  // ...
}));

// stores/auth.store.ts
type AuthStatus = 'checking' | 'verified' | 'rejected' | 'offline';

interface DesktopUserInfo {
  username: string; // Windows 用户名
  employeeNo: string;
  displayName: string;
  departmentName: string;
  permissions: string[];
}

interface AuthState {
  currentUser: DesktopUserInfo | null;
  accessToken: string | null;
  authStatus: AuthStatus;
  verifyDesktopUser: (clientInfo: DesktopClientInfo) => Promise<boolean>;
  clearSession: () => void;
}

export const useAuthStore = create<AuthState>()((set, get) => ({
  currentUser: null,
  accessToken: null,
  authStatus: 'checking',
  
  verifyDesktopUser: async (clientInfo) => {
    set({ authStatus: 'checking' });
    try {
      const res = await api.post('/auth/desktop/verify', clientInfo);
      if (res.code === 200) {
        set({
          currentUser: res.data,
          accessToken: res.data.accessToken,
          authStatus: 'verified',
        });
        return true;
      } else {
        set({ authStatus: 'rejected' });
        return false;
      }
    } catch (e) {
      set({ authStatus: 'offline' });
      return false;
    }
  },
  
  clearSession: () => set({ currentUser: null, accessToken: null, authStatus: 'checking' }),
}));

// stores/settings.store.ts
export const useSettingsStore = create<SettingsState>()((set) => ({
  // 主题: system=跟随系统, light=亮色, dark=暗色
  theme: 'system',
  notifications: true,
  shortcuts: {},
  setTheme: (theme) => set({ theme }),
  setNotifications: (enabled) => set({ notifications: enabled }),
}));

interface SettingsState {
  theme: 'system' | 'light' | 'dark';
  notifications: boolean;
  shortcuts: Record<string, string>;
  setTheme: (theme: 'system' | 'light' | 'dark') => void;
  setNotifications: (enabled: boolean) => void;
}
```

**使用方式：**

```typescript
// 在组件中使用
const MemoList: React.FC = () => {
  const memos = useMemoStore((state) => state.memos);
  const setCurrentMemo = useMemoStore((state) => state.setCurrentMemo);
  // ...
};

// 选择器优化
const currentMemo = useMemoStore((state) => state.currentMemo);
```

### 5.1 系统托盘

```typescript
// desktop/tray/TrayManager.ts
import { TrayIcon } from '@tauri-apps/api/tray';

export class TrayManager {
  async initialize() {
    const tray = await TrayIcon.new({
      tooltip: 'UniComm',
      menu: this.buildMenu(),
      onClick: () => this.showWindow(),
    });
  }

  private buildMenu() {
    return [
      { label: '显示窗口', onClick: () => this.showWindow() },
      { type: 'separator' },
      { label: '退出', onClick: () => this.quit() },
    ];
  }
}
```

```typescript
// services/socket.ts
export class SocketService {
  private ws: WebSocket | null = null;

  connect() {
    this.ws = new WebSocket(WS_URL);
    this.ws.onopen = () => this.startHeartbeat();
    this.ws.onclose = () => this.reconnect();
  }

  onMessage(handler: (data: any) => void) {
    this.ws?.addEventListener('message', (e) => handler(JSON.parse(e.data)));
  }
}
```

## 六、组件抽离检查清单

当需要将组件抽离到 `unicomm-plugin` 时，检查：

- [ ] 不依赖特定业务数据
- [ ] 不依赖特定 API 路径
- [ ] Props 类型明确，无 any
- [ ] 无业务逻辑，所有逻辑通过 Props 传入
- [ ] 无副作用
- [ ] 样式可自定义

### 6.1 业务组件抽离说明

以下组件当前放在 `unicomm-desktop` 中，未来可能抽离：

- `MemoWorkspace` 中的 Markdown 编辑器工具栏
- `MemoWorkspace` 中的 Memo 列表项
- `QuickMemoWindow` 中的轻量编辑器
- 后续实现标签后，再评估 `TagSelector` 是否抽离

**抽离方式：通过 Props + API Adapter 模式**

```
┌─────────────────────────────────────┐
│  unicomm-plugin                      │
│  - MemoEditor (纯展示)              │
│  - Props: content, onChange         │
│  - 无直接 API 调用                   │
└─────────────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────┐
│  API Adapter (features/memo 层)      │
│  - getMemos, createMemo, etc.      │
└─────────────────────────────────────┘
```
