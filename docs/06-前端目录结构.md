## 二、目录职责说明

### 2.1 components/ vs features/

| 目录 | 说明 | 可抽离到 Plugin |
|------|------|----------------|
| components/ui | 基础 UI 组件（Button、Input 等） | ✅ 是 |
| components/layout | 布局组件 | ✅ 部分 |
| components/business | 业务组件（MemoEditor、MemoList 等） | ⚠️ 视情况 |
| features/memo | 功能模块（完整业务逻辑） | ❌ 否 |

### 2.2 pages/ vs features/

| 目录 | 说明 |
|------|------|
| pages/ | 页面容器，只做布局和状态展示 |
| features/ | 功能模块，包含业务逻辑、API、状态 |

### 2.3 stores/ vs features/*/store/

| 目录 | 说明 |
|------|------|
| stores/ | 全局共享状态（认证、主题） |
| features/*/store/ | 功能内专用状态 |

## 三、组件分层架构

```
┌─────────────────────────────────────────────────────────┐
│  Pages（页面层）                                        │
│  职责：布局、路由跳转、错误边界                          │
│  示例：MemoPage.tsx                                     │
├─────────────────────────────────────────────────────────┤
│  Features（功能层）                                     │
│  职责：业务逻辑、数据处理、API 调用                      │
│  示例：useMemos.ts, memoStore.ts, MemoEditor.tsx       │
├─────────────────────────────────────────────────────────┤
│  Components（组件层）                                    │
│  职责：UI 展示、状态管理（useState）、事件处理           │
│  示例：MemoItem.tsx, GroupItem.tsx                       │
├─────────────────────────────────────────────────────────┤
│  UI（基础层）                                           │
│  职责：最底层组件，不含任何业务逻辑                      │
│  示例：Button.tsx, Input.tsx, Modal.tsx                 │
└─────────────────────────────────────────────────────────┘
```

## 五、状态管理规范

### 5.1 状态管理选择

| 场景 | 方案 | 说明 |
|------|------|------|
| 服务端数据 | TanStack Query | 缓存、同步、加载状态 |
| 客户端临时状态 | useState | 不需要全局共享 |
| 跨组件状态 | Zustand | 用户信息、侧边栏状态 |
| 表单状态 | React Hook Form | 复杂表单校验 |

### 5.2 Zustand Store 规范

## 六、API 调用规范

### 6.1 API 文件结构

```typescript
// src/features/memo/api/memoApi.ts

import { request } from '@/services/request';
import type { MemoDTO, MemoVO, PageResult } from '@/types/api';

// 分页查询
export const getMemos = (params: MemoQueryDTO) => {
  return request.get<PageResult<MemoVO>>('/api/v1/memos', { params });
};

// 获取详情
export const getMemoById = (id: number) => {
  return request.get<MemoVO>(`/api/v1/memos/${id}`);
};

// 创建
export const createMemo = (data: MemoCreateDTO) => {
  return request.post<MemoVO>('/api/v1/memos', data);
};

// 更新
export const updateMemo = (id: number, data: MemoUpdateDTO) => {
  return request.put<MemoVO>(`/api/v1/memos/${id}`, data);
};

// 删除
export const deleteMemo = (id: number) => {
  return request.delete(`/api/v1/memos/${id}`);
};
```

### 6.2 TanStack Query 使用规范

```typescript
// ✅ 正确
const { data, isLoading, error } = useQuery({
  queryKey: ['memos', { groupId, keyword }],
  queryFn: () => memoApi.getMemos({ groupId, keyword }),
  staleTime: 1000 * 60 * 5, // 5 分钟内不自动刷新
});

// ❌ 错误：在组件内直接 fetch
useEffect(() => {
  fetch('/api/memos').then(...);
}, []);
```

## 十、组件抽离检查清单

当需要将组件抽离到 `unicomm-plugin` 时，检查：

- [ ] 不依赖特定业务数据（如当前用户信息）
- [ ] 不依赖特定 API 路径
- [ ] Props 类型明确，无 any
- [ ] 无业务逻辑，所有逻辑通过 Props 传入
- [ ] 无副作用（useEffect 内的 API 调用）
- [ ] 样式可自定义（支持 props 或 className）
## 十一、业务组件抽离说明

### 11.1 当前状态

以下组件当前放在 `unicomm-web` 中，不直接抽离到 plugin：
- `MemoEditor` - Markdown 编辑器
- `MemoList` - Memo 列表组件
- `GroupList` - 分组列表组件
- `TagSelector` - 标签选择器

### 11.2 未来抽离方式

未来需要抽离时，通过 **Props + API Adapter** 模式实现：

```
┌─────────────────────────────────────────────────────────┐
│                    unicomm-plugin                       │
│  ┌─────────────────────────────────────────────────┐  │
│  │  MemoEditor / MemoList / GroupList / TagSelector│  │
│  │  - Props: 所有数据通过 props 传入                 │  │
│  │  - Callbacks: 事件通过 onXxx 回调处理            │  │
│  │  - 无直接 API 调用                               │  │
│  └─────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────┐
│                    API Adapter (业务层)                  │
│  ┌─────────────────────────────────────────────────┐  │
│  │  const useMemoAdapter = {                        │  │
│  │    getMemos: () => memoApi.getMemos(),          │  │
│  │    createMemo: (data) => memoApi.create(data),  │  │
│  │    // ...                                       │  │
│  │  };                                             │  │
│  └─────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
```

### 11.3 设计原则

**当前开发时遵守：**
1. 组件内部不直接调用 API，依赖 props 传入数据
2. 事件通过 callbacks（onSave, onDelete 等）向上传递
3. Store 和 API 调用放在 `features/memo` 层
4. 组件保持"纯展示"特性，状态管理在上层

**好处：**
- 未来可按需抽离，不需要大范围重构
- 组件可复用：同一个组件可用于不同数据源
- 便于单元测试
