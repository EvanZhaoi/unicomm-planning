## 二、表详细设计

### 2.1 uni_memo(备忘录主表)

```sql
CREATE TABLE `uni_memo` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键ID',

  -- 内容字段
  `title` VARCHAR(200) NOT NULL DEFAULT '' COMMENT '标题',
  `content` TEXT COMMENT 'Markdown 内容',

  -- 组织和状态
  `group_id` BIGINT NOT NULL DEFAULT 0 COMMENT '分组ID',
  `status` VARCHAR(20) NOT NULL DEFAULT 'normal' COMMENT '状态: normal/todo/done',
  `is_top` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '是否置顶: 0-否 1-是',
  `is_favorite` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '是否收藏: 0-否 1-是',
  `is_archived` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '是否归档: 0-否 1-是(归档不等于状态变化)',

  -- 审计字段
  `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `create_user` BIGINT NOT NULL COMMENT '创建人',
  `deleted` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '逻辑删除: 0-未删 1-已删',
  `deleted_time` DATETIME DEFAULT NULL COMMENT '删除时间(用于回收站)',

  -- 索引
  PRIMARY KEY (`id`),
  -- 列表查询最优索引
  INDEX `idx_user_list` (`create_user`, `deleted`, `is_archived`, `group_id`, `is_top`, `update_time`),
  INDEX `idx_user_status` (`create_user`, `status`),
  INDEX `idx_user_favorite` (`create_user`, `is_favorite`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='备忘录主表';
```

**字段说明:**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键,自增 |
| title | VARCHAR(200) | 标题,最大 200 字符 |
| content | TEXT | Markdown 内容,最大 64KB |
| group_id | BIGINT | 关联分组,默认 0(未分组) |
| status | VARCHAR(20) | 状态:normal(普通)/todo(待办)/done(已完成) |
| is_top | TINYINT | 置顶:0-否 1-是 |
| is_favorite | TINYINT | 收藏:0-否 1-是 |
| is_archived | TINYINT | 归档:0-否 1-是(归档不影响 status) |
| create_time | DATETIME | 创建时间,自动维护 |
| update_time | DATETIME | 更新时间,自动维护 |
| create_user | BIGINT | 创建人,用于数据隔离 |
| deleted | TINYINT | 逻辑删除标识 |
| deleted_time | DATETIME | 删除时间(用于回收站和恢复能力) |

**索引说明:**

| 索引 | 用途 |
|------|------|
| idx_user_list | 列表查询最优:用户+未删+归档+分组+置顶+排序 |
| idx_user_status | 按用户+状态查询 |
| idx_user_favorite | 我的收藏查询 |

### 2.2 uni_memo_group(分组表)

```sql
CREATE TABLE `uni_memo_group` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键ID',

  -- 分组信息
  `name` VARCHAR(50) NOT NULL COMMENT '分组名称',
  `color` VARCHAR(20) NOT NULL DEFAULT '#6B7280' COMMENT '颜色(HEX)',
  `icon` VARCHAR(50) NOT NULL DEFAULT '📁' COMMENT '图标(emoji)',
  `sort_order` INT NOT NULL DEFAULT 0 COMMENT '排序,越小越靠前',

  -- 审计字段
  `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `create_user` BIGINT NOT NULL COMMENT '创建人',
  `is_default` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '是否默认分组: 0-否 1-是',
  `deleted` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '逻辑删除: 0-未删 1-已删',

  -- 索引
  PRIMARY KEY (`id`),
  -- 默认分组查询最优索引
  INDEX `idx_user_default` (`create_user`, `is_default`, `deleted`),
  INDEX `idx_user_sort` (`create_user`, `sort_order`),
  INDEX `idx_deleted` (`deleted`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='备忘录分组表';
```

**字段说明:**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键,自增 |
| name | VARCHAR(50) | 分组名称 |
| color | VARCHAR(20) | 颜色,HEX 格式如 #2563EB |
| icon | VARCHAR(50) | 图标,emoji 或 icon name |
| sort_order | INT | 排序序号 |
| create_time | DATETIME | 创建时间 |
| update_time | DATETIME | 更新时间 |
| create_user | BIGINT | 创建人 |
| is_default | TINYINT | 是否默认分组(0=否,1=是),默认分组不可删除 |
| deleted | TINYINT | 逻辑删除 |

**初始化数据:**

```sql
-- 用户创建时,自动创建默认分组
INSERT INTO `uni_memo_group` (`name`, `create_user`, `is_default`)
VALUES ('我的备忘', #{userId}, 1);
```

**约束:**
- 默认分组(is_default=1)的 name 通常为"我的备忘"
- 默认分组不可删除
- 删除分组时,该组下的 Memo 自动移至默认分组

| tag_id | BIGINT | 标签 ID |

- 同一个 Memo-Tag 组合只能出现一次

## 三、ER 关系图

```
┌─────────────────┐         ┌─────────────────┐
│   uni_memo      │         │ uni_memo_group  │
├─────────────────┤         ├─────────────────┤
│ id (PK)         │         │ id (PK)         │
│ title           │         │ name            │
│ content         │         │ color           │
│ group_id (FK)───┼────────►│ icon            │
│ status          │         │ sort_order      │
│ is_top          │         │ is_default      │
│ is_favorite     │         │ create_user (FK)│
│ is_archived     │         │ deleted         │
│ create_user (FK)│                │
│ deleted         │                │
│ deleted_time    │                │
└───────┬─────────┘                │
        │                         │
        │    ┌────────────────────┘
        │    │ N:M 关系通过中间表
        ▼    ▼
┌─────────────────┐         ┌─────────────────┐
│uni_memo_tag_rel│         │  uni_memo_tag   │
├─────────────────┤         ├─────────────────┤
│ id (PK)         │         │ id (PK)         │
│ memo_id (FK)────┼────────►│ name            │
│ tag_id (FK)─────┼─────────│ color           │
│ create_time     │         │ create_user (FK)│
└─────────────────┘         │ deleted         │

```

**关系说明:**
- Memo : Group = N : 1
- Memo : Tag = N : M(通过关联表)
- Group : User = N : 1
- Tag : User = N : 1

## 四、查询场景与索引优化

### 4.1 场景一:获取用户的 Memo 列表(分页)

```sql
-- 场景:获取"我的备忘"分组下的 Memo,按更新时间倒序
SELECT m.*
FROM uni_memo m
WHERE m.create_user = #{userId}
  AND m.deleted = 0
  AND m.group_id = #{groupId}
ORDER BY m.is_top DESC, m.update_time DESC
LIMIT 20 OFFSET 0;
```

**所需索引：** `idx_user_list`

---

### 4.2 场景二:搜索 Memo

```sql
-- 场景:搜索标题包含"会议"的 Memo
SELECT m.*
FROM uni_memo m
WHERE m.create_user = #{userId}
  AND m.deleted = 0
  AND m.title LIKE '%会议%'
ORDER BY m.update_time DESC;
```

**所需索引：** `idx_user_list`

---

### 4.3 场景三:获取我的收藏

```sql
SELECT m.*
FROM uni_memo m
WHERE m.create_user = #{userId}
  AND m.deleted = 0
  AND m.is_favorite = 1
ORDER BY m.update_time DESC;
```

**所需索引:** `idx_user_favorite`

---

### 4.4 场景四:获取已归档 Memo

```sql
SELECT m.*
FROM uni_memo m
WHERE m.create_user = #{userId}
  AND m.deleted = 0
  AND m.is_archived = 1
ORDER BY m.update_time DESC;
```

**所需索引:** `idx_user_archived`

---

### 4.5 场景五:获取默认分组

```sql
SELECT * FROM uni_memo_group
WHERE create_user = #{userId} AND is_default = 1 AND deleted = 0;
```

**所需索引：** `idx_user_default`

## 六、扩展预留

### 6.1 多租户扩展

未来需要支持多租户时,可添加字段:

```sql
ALTER TABLE `uni_memo` ADD COLUMN `tenant_id` BIGINT NOT NULL DEFAULT 0 COMMENT '租户ID';
```

### 6.2 模块扩展

未来增加 IM/Notify 模块时,可扩展 module_code:

```sql
ALTER TABLE `uni_memo` ADD COLUMN `module_code` VARCHAR(32) NOT NULL DEFAULT 'MEMO' COMMENT '模块编码: MEMO/IM/NOTIFY';
```

### 6.3 恢复功能扩展

deleted_time 字段用于支持回收站和恢复能力:

```sql
-- 查看已删除的 Memo(30天内)
SELECT * FROM uni_memo
WHERE deleted = 1 AND deleted_time > DATE_SUB(NOW(), INTERVAL 30 DAY);

-- 恢复 Memo
UPDATE uni_memo SET deleted = 0, deleted_time = NULL WHERE id = #{id};
```
