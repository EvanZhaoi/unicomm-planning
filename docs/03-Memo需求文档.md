# Memo 功能需求文档

## 一、功能定位

### 1.1 产品定位

**Memo** 是员工个人备忘录系统，用于：

- 工作记录
- 临时事项
- Markdown 笔记
- 个人待办
- 会议记录
- 灵感记录

### 1.2 产品定位约束

| 特性 | 说明 |
|------|------|
| 不是业务系统 Memo | 不绑定 systemId / bizId |
| 不是 IM 附属功能 | 独立模块 |
| 个人使用 | 不涉及团队协作（当前阶段） |

### 1.3 风格参考

整体风格参考：Notion、Craft、Linear、Obsidian（轻量感）

要求：
- 简洁
- 干净
- 高信息密度
- 不花哨
- 可长期扩展

## 二、功能清单

### 2.1 Memo 基础功能

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 新建 Memo | 创建新备忘录 | P0 |
| 编辑 Memo | 修改标题、内容 | P0 |
| 删除 Memo | 逻辑删除 | P0 |
| 查看 Memo | 查看详情 | P0 |
| Markdown 渲染 | 支持 Markdown 内容 | P0 |

### 2.2 组织功能

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 分组管理 | 新建/编辑/删除/排序分组 | P0 |
| 标签功能 | 添加/删除/搜索标签 | P1 |
| 置顶功能 | 将重要 Memo 置顶 | P1 |
| 收藏功能 | 收藏常用 Memo | P1 |
| 归档功能 | 将 Memo 归档 | P1 |

### 2.3 搜索筛选

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 标题搜索 | 按标题关键词搜索 | P0 |
| 内容搜索 | 按内容关键词搜索 | P1 |
| 标签筛选 | 按标签筛选 | P1 |
| 状态筛选 | 筛选 normal/todo/done | P1 |
| 分组筛选 | 按分组筛选 | P0 |

### 2.4 功能优先级汇总

```
P0（必须完成）：
- 新建/编辑/删除/查看 Memo
- 分组管理
- Markdown 内容
- 分组筛选
- 标题搜索

P1（第二阶段）：
- 标签功能
- 置顶/收藏/归档
- 内容搜索
- 状态筛选
```

## 三、数据模型

### 3.1 Memo 实体

```typescript
interface Memo {
  id: number;           // 主键
  title: string;        // 标题
  content: string;      // Markdown 内容
  groupId: number;      // 分组 ID
  status: 'normal' | 'todo' | 'done';  // 状态
  isTop: boolean;       // 是否置顶
  isFavorite: boolean;  // 是否收藏
  isArchived: boolean;  // 是否归档
  tags: string[];       // 标签列表
  createTime: Date;     // 创建时间
  updateTime: Date;     // 更新时间
  createUser: number;   // 创建人
}
```

### 3.2 Memo Group 实体

```typescript
interface MemoGroup {
  id: number;           // 主键
  name: string;        // 分组名称
  color: string;       // 颜色（hex）
  icon: string;        // 图标（emoji 或 icon name）
  sortOrder: number;   // 排序
  createTime: Date;     // 创建时间
  updateTime: Date;     // 更新时间
  createUser: number;   // 创建人
}
```

### 3.3 Memo Tag 实体

```typescript
interface MemoTag {
  id: number;           // 主键
  name: string;        // 标签名称
  color: string;       // 颜色
  createTime: Date;     // 创建时间
  createUser: number;   // 创建人
}
```

### 3.4 关系说明

```
Memo 1:N Group
Memo N:M Tag（通过中间表）
```

## 四、功能详细说明

### 4.1 新建 Memo

**触发方式：**
- 点击左侧"新建"按钮
- 快捷键 Ctrl/Cmd + N

**操作流程：**
1. 点击新建 → 右侧编辑器打开
2. 输入标题（可选，默认"无标题"）
3. 输入 Markdown 内容
4. 选择分组（默认"我的备忘"）
5. 点击保存 或 Ctrl/Cmd + S

**前置条件：**
- 用户已登录

### 4.2 编辑 Memo

**触发方式：**
- 点击列表中的 Memo 项

**操作流程：**
1. 点击 Memo → 右侧编辑器显示内容
2. 修改标题/内容
3. 自动保存（失焦后）或 手动保存

**边界情况：**
- 内容为空时提示"内容不能为空"
- 标题过长时提示"标题最多 200 字符"

### 4.3 删除 Memo

**操作流程：**
1. 点击 Memo 右上角"..."菜单
2. 选择"删除"
3. 确认弹窗
4. 执行逻辑删除（deleted = 1）

**边界情况：**
- 删除后列表刷新
- 已删除的 Memo 可在"回收站"恢复（当前阶段不做 UI）

### 4.4 分组管理

**默认分组：**
- "我的备忘"（默认，不可删除）

**分组操作：**
- 新建分组：点击"+"按钮，输入名称、选择颜色/图标
- 编辑分组：右键分组 → 编辑
- 删除分组：右键分组 → 删除
- 排序分组：拖拽分组排序

**删除分组时的处理：**
- 该分组下的 Memo 自动移动到"我的备忘"
- 不可删除"我的备忘"分组

### 4.5 标签功能

**标签操作：**
- 添加标签：在编辑 Memo 时，点击标签输入框，选择已有标签或新建
- 删除标签：点击标签右侧"x"
- 搜索标签：在标签列表点击标签，筛选相关 Memo

**标签数量限制：**
- 每个 Memo 最多 10 个标签
- 标签名称最多 20 字符

### 4.6 置顶/收藏/归档

**置顶（isTop）：**
- 置顶的 Memo 始终显示在列表顶部
- 支持取消置顶

**收藏（isFavorite）：**
- 左侧导航显示"我的收藏"入口
- 点击后筛选 isFavorite = true 的 Memo

**归档（isArchived）：**
- 归档的 Memo 不在正常列表显示
- 左侧导航显示"已归档"入口
- 支持从归档恢复

### 4.7 搜索功能

**搜索范围：**
- 标题（title LIKE '%keyword%'）
- 内容（content LIKE '%keyword%'）
- 标签（tag.name LIKE '%keyword%'）

**筛选条件：**
- 分组（groupId）
- 状态（status: normal/todo/done）
- 标签（tagIds）
- 时间范围（createTime）

**搜索响应：**
- 实时搜索（debounce 300ms）
- 高亮匹配关键词

## 六、Markdown 支持

### 6.1 支持的 Markdown 语法

| 语法 | 示例 |
|------|------|
| 标题 | # H1, ## H2, ### H3 |
| 列表 | - 无序列表, 1. 有序列表 |
| 任务列表 | - [ ] 待办, - [x] 完成 |
| 引用 | > 引用文本 |
| 代码块 | ```language\n code \n``` |
| 行内代码 | `code` |
| 粗体/斜体 | **bold**, *italic* |
| 链接/图片 | [text](url), ![alt](url) |

### 6.2 不支持的复杂功能

当前阶段不支持：
- 表格（未来扩展）
- 数学公式（未来扩展）
- 思维导图（未来扩展）
- 嵌入文件（未来扩展）

## 七、状态定义

### 7.1 Memo 状态

| 状态 | 值 | 说明 |
|------|-----|------|
| normal | normal | 普通状态 |
| todo | todo | 待办状态 |
| done | done | 已完成状态 |

### 7.2 删除状态

| 状态 | 值 | 说明 |
|------|-----|------|
| 未删除 | 0 | 正常数据 |
| 已删除 | 1 | 逻辑删除 |

## 八、非功能性需求

### 8.1 性能要求

| 指标 | 要求 |
|------|------|
| 首屏加载 | < 1s |
| 列表查询 | < 500ms |
| 保存操作 | < 1s |
| 搜索响应 | < 300ms |

### 8.2 数据安全

- 用户只能访问自己的 Memo
- 逻辑删除，数据可恢复
- 敏感操作需记录日志

### 8.3 兼容性

- **Windows** 为第一优先级
- 后续支持 macOS
- Linux 暂不作为第一优先级
- 响应式布局（最小宽度 1024px）
