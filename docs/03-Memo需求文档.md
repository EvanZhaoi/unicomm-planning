# Memo 功能需求文档

## 一、功能定位

### 1.1 产品定位

**Memo** 是员工备忘录与轻量共享系统，用于：

- 工作记录
- 临时事项
- Markdown 笔记
- 个人待办
- 会议记录
- 灵感记录

### 1.2 产品定位约束

| 特性 | 说明 |
|------|------|
| 不是业务系统 Memo | 不绑定 systemId / bizId |
| 不是 IM 附属功能 | 独立模块 |
| 个人拥有，相关人可见 | Memo 有明确创建者，创建者可以添加相关人，相关人默认仅可查看 |

### 1.3 风格参考

整体风格参考：Notion、Craft、Linear、Obsidian（轻量感）

要求：
- 简洁
- 干净
- 高信息密度
- 不花哨
- 可长期扩展

## 二、功能清单

### 2.1 Memo 基础功能

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 新建 Memo | 创建新备忘录 | P0 |
| 快速新增 Memo | 通过全局快捷键唤出简易窗口，仅输入内容并新增 | P0 |
| 编辑 Memo | 修改标题、内容 | P0 |
| 删除 Memo | 逻辑删除 | P0 |
| 查看 Memo | 查看详情 | P0 |
| 相关人查看/编辑 | 编辑时添加相关人，并按 view/edit 权限控制查看或编辑 | P0 |
| 与我相关 | 仅查看别人共享给当前用户的 Memo | P0 |
| 实时刷新 | Memo 变更通过 WebSocket 推送给创建者和相关人，桌面端自动刷新 | P0 |
| 富文本编辑与 MD 源码 | 默认可视化编辑，必要时切换 MD 源码 | P0 |

### 2.4 功能优先级汇总

```

- 置顶/收藏

```

## 三、数据模型

### 3.1 Memo 实体

```typescript
interface Memo {
  id: number;           // 主键
  title: string;        // 标题
  content: string;      // Markdown 内容
  groupId: number;      // 分组 ID
  status: 'normal' | 'todo' | 'done';  // 状态
  isTop: boolean;       // 是否置顶
  isFavorite: boolean;  // 当前用户是否收藏
  createTime: Date;     // 创建时间
  updateTime: Date;     // 更新时间
  ownerUsername: string;   // 所有者用户名
  isOwner: boolean;        // 当前用户是否创建者
  isShared: boolean;       // 当前用户是否通过相关人关系查看
  relatedUsers: MemoRelatedUser[]; // 相关人列表
}
```

### 3.1.1 Memo Related User 实体

```typescript
interface MemoRelatedUser {
  id: number;
  memoId: number;
  ownerUsername: string;
  username: string;        // 相关人用户名
  permission: 'view';      // 当前阶段仅支持查看，后续可扩展 edit
  createTime: Date;
  updateTime: Date;
}
```

### 3.2 Memo Group 实体

```typescript
interface MemoGroup {
  id: number;           // 主键
  name: string;        // 分组名称
  color: string;       // 颜色（hex）
  icon: string;        // 图标（emoji 或 icon name）
  sortOrder: number;   // 排序
  createTime: Date;     // 创建时间
  updateTime: Date;     // 更新时间
  ownerUsername: string;   // 所有者用户名
}
```

### 3.4 关系说明

```
Memo 1:N Group
Memo 1:N RelatedUser（创建者添加相关人，相关人按 view/edit 权限查看或编辑）
```

## 四、功能详细说明

### 4.1 新建 Memo

**触发方式：**
- 点击左侧"新建"按钮
- 快捷键 Ctrl/Cmd + N
- 通过快速 Memo 全局快捷键唤出简易窗口新增

**操作流程：**
1. 点击新建 → 右侧编辑器打开
2. 输入标题（可选；未输入时仅显示"请输入标题"占位符，不写入默认标题）
3. 输入 Markdown 内容
4. 选择分组（默认"我的备忘"）
5. 点击保存 或 Ctrl/Cmd + S

**当前实现说明：**
- 新建 Memo 会使用当前分组筛选作为默认分组
- 如果当前处于状态筛选，新增 Memo 会使用当前筛选状态
- 当前阶段为手动保存，底部保存按钮调用更新接口

**前置条件：**
- UniComm 桌面端已完成 Windows 当前用户识别，并通过后端身份校验

### 4.1.1 快速 Memo

**功能定位：**

快速 Memo 是桌面端高频入口，用于员工在任何应用中快速记录临时事项，不进入完整 Memo 工作台。

**触发方式：**
- 全局快捷键唤出，默认 `Ctrl + Alt + N`
- 后续可扩展为托盘菜单入口

**界面范围：**
- 仅显示一个轻量编辑器
- 可选择分组
- 仅支持新增 Memo
- 不支持编辑已有 Memo
- 不支持删除、置顶、收藏等管理操作

**操作流程：**
1. 用户按下快速 Memo 快捷键
2. 桌面端唤出简易 Memo 小窗口
3. 用户输入内容
4. 系统取第一行非空内容作为标题，正文保存到 content
5. 点击"新增"或使用提交快捷键保存
6. 保存成功后隐藏简易窗口
7. 主 Memo 列表收到新增事件后刷新

**边界情况：**
- 内容为空时不可保存
- 快捷键被其他应用占用时，设置页提示保存失败
- 用户未认证完成时，简易窗口显示初始化状态，不允许新增

### 4.1.2 快捷键设置

**设置位置：** 桌面端设置页面

| 快捷键 | 默认值 | 说明 |
|--------|--------|------|
| 唤出主界面 | `Ctrl + Alt + M` | 显示主窗口并聚焦 |
| 唤出快速 Memo | `Ctrl + Alt + N` | 显示简易 Memo 小窗口并聚焦编辑器 |

**设置规则：**
- 两个快捷键不能为空
- 两个快捷键不能相同
- 保存后立即重新注册全局快捷键
- 设置持久化到本地配置
- 注册失败时保留原快捷键，并提示错误

### 4.2 编辑 Memo

**触发方式：**
- 点击列表中的 Memo 项

**操作流程：**
1. 点击 Memo → 右侧编辑器显示内容
2. 修改标题/内容
3. 默认在可视化编辑器中直接编辑，编辑效果与阅读效果一致
4. 通过工具栏插入格式或图片
5. 如需修改 Markdown，切换到 MD 源码视图；如需对照修改，切换到双栏视图
5. 点击保存

### 4.2.1 添加相关人

相关人用于让指定员工查看或协作编辑该 Memo。创建者可以为每个相关人设置 `view` 或 `edit` 权限；创建者仍是唯一可以删除、置顶、调整分组和管理相关人权限的人。收藏是每个用户自己的关系状态，创建者和相关人都可以收藏自己可见的 Memo。

1. 创建者打开 Memo 编辑界面
2. 在相关人区域输入姓名、用户名或工号进行模糊搜索
3. 从搜索结果中选择成员，界面以胶囊项展示姓名和工号
4. 保存 Memo
5. 后端替换该 Memo 的相关人列表和权限
6. WebSocket 向创建者和相关人推送 `memo.updated` 或 `memo.related.updated`
7. 相关人桌面端收到事件后刷新列表，可看到该 Memo

**约束：**

- 创建者不能把自己添加为相关人
- 重复用户名会自动去重
- `view` 权限只能查看 Memo；`edit` 权限可编辑标题、正文和状态
- 只有创建者可以调整分组、相关人、置顶和删除
- 收藏按用户隔离记录，相关人收藏不会影响创建者或其他相关人
- 桌面端需要在列表和编辑区展示当前用户对共享 Memo 的权限，避免把只读/可写状态藏在禁用按钮里
- 当前阶段不做多人同时光标协同编辑，实时同步以保存后的 WebSocket 事件刷新为主
- 成员搜索接口当前支持姓名、用户名、工号、邮箱模糊搜索
- 前端成员搜索使用通用 `RemoteMultiSelect`，输入框、移除按钮和弹出结果列表应复用基础 UI 组件，不在业务组件里重复手写远程多选交互

**权限矩阵：**

| 操作 | 创建者 owner | 相关人 edit | 相关人 view |
|------|--------------|-------------|-------------|
| 查看 Memo | 是 | 是 | 是 |
| 编辑标题/正文/状态 | 是 | 是 | 否 |
| 调整分组 | 是 | 否 | 否 |
| 添加/移除相关人 | 是 | 否 | 否 |
| 修改相关人权限 | 是 | 否 | 否 |
| 置顶 | 是 | 否 | 否 |
| 收藏/取消收藏 | 是 | 是 | 是 |
| 删除 Memo | 是 | 否 | 否 |

**边界情况：**
- 内容为空时提示"内容不能为空"
- 标题过长时提示"标题最多 200 字符"

### 4.3 删除 Memo

**操作流程：**
1. 点击 Memo
2. 在底部操作栏点击"删除"
3. 确认弹窗
4. 执行逻辑删除（deleted = 1）

**边界情况：**
- 删除后列表刷新
- 已删除的 Memo 可在"回收站"恢复（当前阶段不做 UI）

### 4.4 分组管理

**默认分组：**
- "我的备忘"（默认，不可删除）

**分组操作：**
- 新建分组：点击分组标题右侧 "+" 按钮，输入名称、选择颜色/图标
- 编辑分组：点击具体分组右侧编辑按钮，修改名称、颜色和图标
- 删除分组：在分组编辑弹层中点击删除按钮，并通过确认弹窗二次确认
- 排序分组：拖拽分组排序

**删除分组时的处理：**
- 该分组下的 Memo 自动移动到"我的备忘"
- 不可删除"我的备忘"分组

### 4.6 置顶/收藏

**置顶（isTop）：**
- 置顶的 Memo 始终显示在列表顶部
- 支持取消置顶
- 当前桌面端在编辑器底部操作栏提供"置顶/取消置顶"入口

**收藏（isFavorite）：**
- 左侧导航显示"我的收藏"入口
- 点击后筛选当前用户收藏的 Memo
- 当前桌面端在编辑器底部操作栏提供"收藏"入口
- 收藏关系按 `username + memoId` 记录，不能存放在 Memo 主表中

### 4.7 搜索功能

**搜索范围：**
- 标题（title LIKE '%keyword%'）
- 内容（content LIKE '%keyword%'）

**筛选条件：**
- 分组（groupId）
- 状态（status: normal/todo/done）
- 时间范围（createTime）

**搜索响应：**
- 当前阶段：输入关键词后按 Enter 执行查询
- 后续阶段：实时搜索（debounce 300ms）与关键词高亮

## 六、编辑器设计

### 6.1 编辑器选型

> **重要：** 当前 Memo 编辑器不再只面向会 Markdown 的用户。
> 公司大部分用户不是程序员，因此编辑器必须支持普通用户友好的编辑方式。

**当前技术方案：** Markdown 存储 + Milkdown Crepe 双栏编辑器（左侧可视化编辑，右侧 MD 源码同步）。

**组件许可：** Milkdown / Crepe 使用 MIT 协议，符合开源免费商用要求。

**选型理由：**
- 用户体验更接近富文本（类似 Word / 飞书文档 / Notion）
- 内部存储为 Markdown 或 Markdown-compatible 格式
- 可扩展性强（后续可加上传、AI、协作、快捷命令）
- 默认可视化编辑，降低普通用户学习成本

### 6.2 编辑模式

| 模式 | 说明 | 适用用户 |
|------|------|----------|
| 可视化编辑 | 单独显示所见即所得编辑器 | 默认模式 / 普通员工 |
| Markdown 源码 | 单独显示源码编辑器 | 程序员/高级用户 |
| 双栏编辑 | 左侧所见即所得，右侧 Markdown 源码同步 | 对照编辑 / 宽屏 |
| 预览 | 与可视化编辑合并，编辑区即预览效果 | 所有人 |

**当前默认模式：** 可视化编辑

**后续目标默认模式：** 可视化编辑

### 6.3 支持的格式化功能

| 功能 | 说明 | 工具栏按钮 |
|------|------|------------|
| 普通文本输入 | 基础文本 | - |
| 标题 | 当前支持 H1 插入，后续扩展 H2 / H3 | H1 |
| 加粗 | 粗体文字 | **B** |
| 斜体 | 斜体文字 | *I* |
| 删除线 | 删除线文字 | S |
| 列表 | 无序列表 | 列表 |
| 有序列表 | 数字编号列表 | 1.列表 |
| 待办 checkbox | 任务列表 | 待办 |
| 引用 | 引用文本块 | 引用 |
| 代码块 | 代码片段（支持语言） | 代码 |
| 行内代码 | 行内代码 | 代码 |

### 6.4 编辑器工具栏

```
Milkdown Crepe 内置浮动工具栏、块级菜单、图片菜单 | 可视化 | MD 源码
```

### 6.5 编辑器状态栏

编辑器底部显示：
- 当前保存状态（已保存 / 保存失败）
- 操作按钮（置顶、收藏、保存、删除）
- 后续可扩展字数统计和最后保存时间

### 6.6 当前技术实现说明

- 默认显示 Milkdown Crepe 可视化编辑器；模式切换顺序为“可视化 / MD 源码 / 双栏”
- 内容保存仍使用 Markdown 文本，便于兼容历史数据和后续导出
- 可视化模式下由 Crepe 提供 TopBar、浮动工具栏、块级菜单、表格、代码块、公式、图片插入等能力
- 双栏模式下，右侧 MD 源码修改后同步回左侧可视化编辑器，左侧可视化修改后同步回源码区
- 图片上传回调当前读取本地图片为 base64，并以 Markdown 图片语法 `![image](data:image/...)` 存入 content
- 后续如果接入服务端上传接口，可以将图片上传回调从 base64 写入替换为上传后插入 URL

### 6.7 后续技术实现说明

编辑器后续增强方向：
- 接入服务端文件上传接口，图片保存 URL 而不是 base64
- 根据实际使用情况开启表格、代码高亮、数学公式等 Crepe 扩展能力
- 数据存储继续使用 Markdown 格式（TEXT 类型，最大 64KB），保持兼容性

### 6.8 原型设计更新

编辑器区域原型需要更新：
- 当前阶段默认显示可视化编辑器，双栏编辑作为最后一个模式选项
- 工具栏使用 Crepe 内置交互，不再维护自研 DOM 工具栏
- 当前支持可视化/MD 源码/双栏切换

## 八、非功能性需求

### 8.1 性能要求

| 指标 | 要求 |
|------|------|

| 列表查询 | < 500ms |
| 保存操作 | < 1s |

### 8.2 数据安全

- 用户可以访问自己创建的 Memo，以及别人共享给自己的 Memo
- “与我相关”只展示别人共享给当前用户的 Memo，不包含自己创建的 Memo
- 创建者拥有编辑、删除、置顶和维护相关人的权限
- 相关人按 `view/edit` 权限查看或编辑共享 Memo
- 收藏是当前用户自己的状态，创建者与相关人的收藏互不影响
- 逻辑删除，数据可恢复
- 敏感操作需记录日志

### 8.3 兼容性

- **Windows** 为第一优先级
- 后续支持 macOS
- Linux 暂不作为第一优先级
- 响应式布局（最小宽度 1024px）

### 8.4 主题切换

 Memo 支持亮色/暗色主题切换：

| 模式 | 说明 |
|------|------|
| 跟随系统 | 自动检测操作系统主题偏好 |
| 亮色模式 | 强制使用亮色主题 |
| 暗色模式 | 强制使用暗色主题 |

**设置位置：** 桌面端设置页面

**实现方式：**

- 使用 CSS 变量管理主题颜色
- 通过 `prefers-color-scheme` 检测系统偏好
- 用户选择覆盖系统偏好，存入本地设置
- 下次启动时优先读取用户设置

### 8.5 语言与字体

Memo 前端界面需要支持中文和日文两种语言，中文界面使用阿里巴巴普惠体 3.0，日文界面使用 Alibaba Sans JP。

| 项目 | 要求 |
|------|------|
| 默认语言 | 中文 `zh-CN` |
| 可选语言 | 中文 `zh-CN` / 日文 `ja-JP` |
| 设置位置 | 桌面端设置页面 |
| 默认语言选择 | 首次启动根据系统语言自动选择，未支持语言回退中文 |
| 中文首选字体 | 阿里巴巴普惠体 3.0 |
| 日文首选字体 | Alibaba Sans JP |
| 字体文件 | 桌面端内置 Regular / Medium / Bold `.woff2` |
| 中文字体栈 | 阿里巴巴普惠体 3.0 + 中文系统字体回退 |
| 日文字体栈 | Alibaba Sans JP + 阿里巴巴普惠体 3.0 + 日文系统字体回退 |

**实现要求：**

- 语言设置持久化到本地设置
- 首次启动且没有本地语言设置时，根据系统语言自动选择默认语言
- `zh-*` 映射为 `zh-CN`，`ja-*` 映射为 `ja-JP`，其他未支持语言回退 `zh-CN`
- 用户手动切换语言后，以用户本地设置为准
- 切换语言后主界面、Memo 工作台、快速 Memo、设置页文案即时更新
- 切换语言后同步更新 `<html lang>` 和 `data-language`
- 字体文件随桌面端前端资源打包，优先从 `src/assets/fonts/` 加载
- 若字体文件加载失败，允许回退到对应语言的系统字体
