## 二、桌面端技术详解

### 2.1 Tauri 2

**选型理由：**

- Rust 后端，性能优异
- WebView2 渲染，无 Chromium 包袱
- 体积小（< 10MB 安装包）
- 支持系统托盘、多窗口
- 支持全局快捷键
- 内置自动更新
- 本地文件系统和剪贴板 API
- 支持 WebSocket

**核心能力：**

```typescript
// 窗口管理
import { getCurrentWindow } from '@tauri-apps/api/window';
const win = getCurrentWindow();
await win.setAlwaysOnTop(true);
await win.minimize();

// 系统托盘
import { TrayIcon } from '@tauri-apps/api/tray';
// 全局快捷键（推荐由 Rust 侧注册，前端仅通过设置页传入配置）
import { invoke } from '@tauri-apps/api/core';
await invoke('configure_global_shortcuts', {
  mainShortcut: 'Ctrl+Alt+M',
  quickMemoShortcut: 'Ctrl+Alt+N',
});
// 通知
import { sendNotification } from '@tauri-apps/plugin-notification';
// 文件系统
import { readTextFile, writeTextFile } from '@tauri-apps/plugin-fs';
```

### 2.2 React + TypeScript

**选型理由：**

- 成熟的 UI 生态
- Hooks API 简化状态管理
- TypeScript 保证类型安全
- 组件化开发模式
- 与 Tauri 集成良好

**编码规范：**

```typescript
// 正确：使用函数组件 + Hooks
const MemoList: React.FC<Props> = ({ memos, onSelect }) => {
  const [selectedId, setSelectedId] = useState<string | null>(null);
  // ...
};

// 禁止：class component
class MemoList extends React.Component { ... }
```

### 2.3 Zustand（状态管理）

**选型理由：**

- 轻量级（比 Redux 简单）
- 足够满足状态管理需求
- TypeScript 支持好
- 支持持久化

**按领域拆分 Store：**

```typescript
// stores/memo.store.ts
interface MemoStore {
  memos: Memo[];
  currentMemo: Memo | null;
  setCurrentMemo: (memo: Memo | null) => void;
}

// stores/socket.store.ts
interface SocketStore {
  connected: boolean;
  reconnecting: boolean;
  connect: () => void;
  disconnect: () => void;
}

// stores/session.store.ts
interface SessionStore {
  user: User | null;
  token: string | null;
  login: (user: User, token: string) => void;
  logout: () => void;
}

// stores/settings.store.ts
interface SettingsStore {
  theme: 'light' | 'dark' | 'system';
  notifications: boolean;
  setTheme: (theme: 'light' | 'dark' | 'system') => void;
}
```

### 2.4 服务端数据管理

当前 Memo 模块使用 `TanStack Query + Axios + Zustand Store` 管理服务端数据。

现有策略：

- API 调用统一放在 `features/*/api`。
- HTTP 基础设施统一放在 `core/http`。
- TanStack Query 负责服务端缓存、同 key 请求去重、短期缓存和失效刷新。
- Zustand 负责 Memo 列表投影、分页游标、选中项、保存状态、编辑草稿等 UI 状态。
- 列表首屏分页加载，滚动到底部再拉取下一页。
- WebSocket 只推送变更事件，最终数据仍通过 HTTP API 拉取。

迁移原则：

- 不直接把现有 Zustand 全部删掉。
- 先用 `queryClient.fetchQuery` 替换手写请求复用。
- mutation 成功后通过 `setQueryData` 和 `invalidateQueries` 更新缓存。
- 待 MemoWorkspace 拆分完成后，再逐步在页面组件中使用 `useQuery`。

**使用示例：**

```typescript
// 正确
const { data, isLoading, error } = useQuery({
  queryKey: ['memos', { groupId, keyword }],
  queryFn: () => memoApi.list({ groupId, keyword }),
  staleTime: 1000 * 60 * 5,
});

// 与 WebSocket 配合：数据更新后主动 invalidate
const queryClient = useQueryClient();
socket.on('memo:updated', () => {
  queryClient.invalidateQueries({ queryKey: ['memos'] });
});
```

### 2.5 TailwindCSS

**选型理由：**

- 原子化 CSS，快速开发
- 减少 CSS 文件体积
- 一致性高
- 支持深色模式

**配置：**

```js
  darkMode: 'class',

    extend: {
      colors: {
        primary: '#2563EB',
      }
    }
  }
}
```

### 2.6 字体与国际化

**字体要求：**

- 前端中文界面首选字体使用阿里巴巴普惠体 3.0
- 前端日文界面首选字体使用 Alibaba Sans JP
- 常用 UI 字重内置到 `src/assets/fonts/`
- 根据设置页语言选择同步设置 `<html lang>` 和 `data-language`

**字体栈示例：**

```css
:root {
  --font-zh: "Alibaba PuHuiTi 3.0", "Microsoft YaHei UI", "Microsoft YaHei", "PingFang SC", sans-serif;
  --font-ja: "Alibaba Sans JP", "Alibaba PuHuiTi 3.0", "Hiragino Sans", "Yu Gothic UI", "Yu Gothic", "Meiryo", "Noto Sans JP", sans-serif;
}
```

**语言要求：**

| 语言 | code | 说明 |
|------|------|------|
| 中文 | zh-CN | 默认语言 |
| 日文 | ja-JP | 设置页可切换 |

首次启动时通过系统语言推导默认界面语言：`zh-*` 映射为 `zh-CN`，`ja-*` 映射为 `ja-JP`，未支持语言回退 `zh-CN`。用户手动切换语言后，以本地保存设置为准。

当前阶段使用轻量自研 i18n 映射维护前端文案，后续模块增多后可再评估 `i18next` 等成熟方案。

## 三、后端技术详解

### 3.1 Spring Boot 4.0.6

**选型理由：**

- Java 21 支持（虚拟线程、Pattern Matching）
- 简化配置
- 内嵌 Tomcat/Jetty
- 生态成熟

### 3.2 Spring WebSocket

**用途：**

- 实时消息推送
- 在线状态同步
- 通知推送

**实现方式：**

当前实现使用 Spring WebSocket 的 `TextWebSocketHandler` 和 `HandshakeInterceptor`：

- 握手阶段校验 Sa-Token。
- 连接级保存 username。
- 广播时按 owner 和相关人过滤接收连接。
- 后端只推送事件，前端收到事件后重新拉取 HTTP 数据。

### 3.3 Sa-Token 认证

**选型理由：**

- 轻量级
- 支持 Token 认证
- 支持分布式 Session
- 权限鉴定灵活
