# UniComm 技术栈方案

## 一、技术选型总览

| 层级 | 技术 | 版本 | 说明 |
|------|------|------|------|
| **后端框架** | Spring Boot | 3.2+ | 最新稳定版，支持 Java 21 |
| **语言** | Java | 21 | LTS 版本，性能最优 |
| **构建工具** | Maven | 3.9+ | 依赖管理、构建 |
| **数据库** | MySQL | 8.0+ | 关系型数据存储 |
| **ORM** | MyBatis Plus | 3.5+ | 简化 CRUD、Plus 提供增强 |
| **缓存** | Redis | 7.0+ | 热点数据、Session |

| **认证** | Sa-Token | 1.37+ | 轻量级、可持续扩展 |
| **文档** | SpringDoc OpenAPI | 2.3+ | Swagger 3.0 规范 |
| **前端框架** | React | 18.x | 主流、Hooks |
| **构建工具** | Vite | 5.x | 快速构建 |

| **状态管理** | Zustand | 4.x | 轻量级 |

## 二、后端技术详解

### 2.1 Spring Boot 3

**选型理由：**
- Java 21 支持（虚拟线程、Pattern Matching）
- 简化配置（无需 web.xml）
- 内嵌 Tomcat/Jetty
- 生态成熟

**版本控制：**
```xml
<java.version>21</java.version>
<spring-boot.version>3.2.5</spring-boot.version>
```

### 2.2 MyBatis Plus

**选型理由：**
- 强大的 CRUD 封装
- 内置分页插件
- 自动填充创建/更新时间
- 逻辑删除支持
- 代码生成器

**核心配置：**
```yaml
mybatis-plus:
  mapper-locations: classpath*:/mapper/**/*.xml
  type-aliases-package: com.unicomm.module.*.entity
  global-config:
    db-config:
      logic-delete-field: deleted
      logic-delete-value: 1
      logic-not-delete-value: 0
  configuration:
    map-underscore-to-camel-case: true
```

### 2.3 Sa-Token 认证

**选型理由：**
- 轻量级（核心包仅 300KB）
- 支持多种认证模式（Cookie、Token、OAuth2）
- 支持分布式 Session
- 权限鉴定灵活
- 文档完善

**认证模式：**
```
当前阶段：Token 模式（最简单的 JWT 类似方案）
未来扩展：OAuth2 模式（对接企业 SSO）
```

**核心功能：**
- 登录认证
- 路由鉴权
- 方法级鉴权
- Session 管理
- 多账号体系

### 2.4 SpringDoc OpenAPI

**选型理由：**
- Swagger 3.0 规范
- 基于注解自动生成文档
- 支持在线调试
- 响应式文档

**使用方式：**
```java
@RestController
@RequestMapping("/api/memo")
@Tag(name = "Memo", description = "备忘录接口")
public class MemoController {
    @Operation(summary = "创建备忘录")
    @PostMapping
    public Result<MemoVO> create(@RequestBody @Valid MemoCreateDTO dto) {
        // ...
    }
}
```

## 三、前端技术详解

### 3.1 React 18

**选型理由：**
- Hooks API 完全取代 class component
- Concurrent Features（未来可升级）
- 生态成熟、组件丰富

**编码规范：**
```typescript
// ✅ 正确：使用函数组件 + Hooks
const MemoList: React.FC<Props> = ({ memos, onSelect }) => {
  const [selectedId, setSelectedId] = useState<string | null>(null);
  const queryClient = useQueryClient();
  
  // ...
};

// ❌ 禁止：class component
class MemoList extends React.Component { ... }
```

### 3.2 TypeScript

**选型理由：**
- 类型安全，开发时发现错误
- IDE 智能提示
- 代码可读性、可维护性

**strict 模式：**
```json
{
  "compilerOptions": {
    "strict": true,
    "noImplicitAny": true,
    "strictNullChecks": true
  }
}
```

### 3.3 TanStack Query

**选型理由：**
- 服务端状态管理（列表、详情）
- 自动缓存 + 后台刷新
- 加载状态、错误处理
- 分页、无限滚动支持

**使用示例：**
```typescript
// ✅ 正确
const { data, isLoading, error } = useQuery({
  queryKey: ['memos', { groupId, keyword }],
  queryFn: () => memoApi.list({ groupId, keyword }),
  staleTime: 1000 * 60 * 5, // 5分钟内不刷新
});

// ❌ 禁止：在组件内直接 fetch
useEffect(() => {
  fetch('/api/memos').then(...); // 难以复用、无法缓存
}, []);
```

### 3.4 Zustand

**选型理由：**
- 轻量级（比 Redux 简单）
- 足够满足状态管理需求
- TypeScript 支持好

**使用场景：**
```typescript
// ✅ 正确：跨组件共享的状态
interface AuthStore {
  user: User | null;
  token: string | null;
  login: (user: User, token: string) => void;
  logout: () => void;
}

// ❌ 禁止：过度使用（能用 useState 就不要用 store）
```

### 3.5 TailwindCSS

**选型理由：**
- 原子化 CSS，快速开发
- 配合设计系统（shadcn/ui）
- 减少 CSS 文件体积
- 一致性高

**配置：**
```js
// tailwind.config.js
module.exports = {
  content: ['./index.html', './src/**/*.{js,ts,jsx,tsx}'],
  theme: {
    extend: {
      colors: {
        primary: '#2563EB',
      }
    }
  }
}
```

### 3.6 React Markdown

**选型理由：**
- 轻量级 Markdown 渲染
- 支持 GFM（GitHub Flavored Markdown）
- 可扩展组件（代码高亮、Todo list）

**使用示例：**
```tsx
import ReactMarkdown from 'react-markdown';
import remarkGfm from 'remark-gfm';

<ReactMarkdown remarkPlugins={[remarkGfm]}>
  {content}
</ReactMarkdown>
```

## 五、技术风险与备选方案

| 风险 | 应对方案 |
|------|----------|
| Java 21 兼容性 | 当前先用 21 LTS，稳定后考虑 22 |
| MyBatis Plus 付费版冲突 | 使用开源版（3.5.x），避免高级特性 |
| Sa-Token 性能问题 | 后续可用 Redis Session 扩展 |
| 前端构建速度 | Vite 已足够快，暂时不考虑 Turborepo |
| Markdown 编辑器选型 | 第一阶段用简单渲染，不做复杂编辑器 |
