# UniComm 前端页面显示规范（视觉篇）

> **状态：** 初稿
> **目的：** 解决现有页面"不好看"的问题，提升视觉品质至 Notion / Linear / Craft 级别
> **适用范围：** unicomm-desktop 所有页面组件

---

## 一、视觉差距分析

### 1.1 当前问题清单

| 问题 | 位置 | 严重程度 |
|------|------|----------|
| 页面缺乏层次感，全平铺效果 | 全局 | 高 |
| 侧边栏过于单薄，无视觉重量 | AppLayout | 高 |
| Memo 编辑器像 textarea，无编辑氛围 | MemoWorkspace | 高 |
| 设置页像后台表单，无精致感 | SettingsPanel | 中 |
| 加载/空状态只有文字，无图形表达 | 全局 | 中 |
| 无微交互动（hover/click 无反馈） | 全局 | 中 |
| 三栏布局无呼吸感，间距过紧 | MemoWorkspace | 中 |
| 认证状态页过于简陋 | AuthStatusView | 中 |
| Windows 原生标题栏与原型标题栏重复 | AppLayout / Tauri Window | 高 |

### 1.2 目标效果参考

**Notion** — 干净、留白充足、信息层次分明
**Linear** — 极简、高信息密度、微妙的渐变和阴影
**Craft** — 精致的卡片、柔和的圆角、舒适的行间距

## 二、通用视觉原则

### 2.1 页面层次

**三层结构：背景层 → 卡片层 → 交互层**

```css
/* 背景层：最底层，全局通用 */
--background: #f8f9fa;        /* 浅色模式下微微暖灰，非纯白 */

/* 卡片层：承载内容的卡片 */
--card: #ffffff;
--card-shadow: 0 1px 3px rgba(0,0,0,0.04), 0 1px 2px rgba(0,0,0,0.06);

/* 深色模式层次 */
.dark {
  --background: #0f0f11;
  --card: #18181b;
  --card-shadow: 0 1px 3px rgba(0,0,0,0.3);
}
```

**禁止：**
- 全部元素平铺在同一层级
- 直接在 `bg-background` 上堆叠列表项
- 所有卡片都用同样的背景色

### 2.2 间距系统

**基础单位：4px**

| 名称 | 值 | 场景 |
|------|-----|------|
| `space-xs` | 4px | 紧凑元素内部 |
| `space-sm` | 8px | 相关元素之间 |
| `space-md` | 16px | 组件内部 |
| `space-lg` | 24px | 区块之间 |
| `space-xl` | 32px | 大区块之间 |
| `space-2xl` | 48px | 页面级留白 |

**MemoWorkspace 当前布局规范：**
```
左侧应用导航 (220px) → 1px 边框 ← Memo 列表 (320px) → 1px 边框 ← 编辑器 (minmax(0,1fr))
```
主窗口总高度由 Tauri 无原生装饰窗口提供，应用内标题栏高度 48px，列表和编辑器高度独立滚动。

### 2.3 圆角系统

| 名称 | 值 | 场景 |
|------|-----|------|
| `radius-sm` | 6px | 小标签、小按钮 |
| `radius-md` | 8px | 输入框、中等卡片 |
| `radius-lg` | 12px | 主卡片、面板 |
| `radius-xl` | 16px | 大模态框 |

**禁止：**
- radius-sm 用于主内容卡片（显得廉价）
- radius-lg 用于按钮（过于圆润）
- 混用不同圆角层级导致视觉混乱

### 2.4 阴影系统

**三层阴影：**

```css
/* 轻阴影：微微浮起，用于卡片、hover 状态 */
--shadow-sm: 0 1px 2px rgba(0,0,0,0.04);

/* 中阴影：浮起效果，用于下拉菜单、popover */
--shadow-md: 0 4px 6px -1px rgba(0,0,0,0.07), 0 2px 4px -2px rgba(0,0,0,0.05);

/* 重阴影：浮起效果，用于模态框 */
--shadow-lg: 0 10px 15px -3px rgba(0,0,0,0.08), 0 4px 6px -4px rgba(0,0,0,0.05);
```

**使用场景：**
- 侧边栏：`shadow-sm`
- 卡片 hover：`shadow-sm + scale(1.01)`
- 下拉菜单：`shadow-md`
- 弹窗/模态框：`shadow-lg`

## 三、颜色系统

### 3.1 颜色层级

页面从背景到前景分为 4 个视觉层级：

```
第1层：背景              → 最弱的存在感（--background）
第2层：次级区域          → 分区分割（--muted, --secondary）
第3层：卡片/面板         → 承载内容（--card, --popover）
第4层：主色调/强调色     → 吸引注意力（--primary, --accent）
```

**当前问题：** 浅色模式下背景 `#ffffff` 与卡片 `#ffffff` 完全相同，无层次区分。

### 3.2 浅色模式颜色（修复后）

```css
:root {
  /* 第1层：背景 — 微微暖灰，区别于纯白 */
  --background: #f8f9fa;
  --foreground: #1a1a1a;

  /* 第2层：次级区域 — 明显区分背景 */
  --muted: #f1f3f5;
  --muted-foreground: #868e96;
  --secondary: #e9ecef;
  --secondary-foreground: #495057;

  /* 第3层：卡片 — 纯白，与背景形成层次 */
  --card: #ffffff;
  --card-foreground: #1a1a1a;
  --popover: #ffffff;
  --popover-foreground: #1a1a1a;

  /* 第4层：交互色 */
  --primary: #2563eb;          /* 主蓝 */
  --primary-foreground: #ffffff;
  --accent: #f1f3f5;           /* 微妙的 hover 区 */
  --accent-foreground: #1a1a1a;

  /* 边框 — 更轻，不突兀 */
  --border: #e9ecef;
  --input: #e9ecef;
  --ring: #2563eb;

  /* 功能色 */
  --destructive: #ef4444;
  --destructive-foreground: #ffffff;
}
```

### 3.3 深色模式颜色（修复后）

```css
.dark {
  /* 第1层：背景 — 深灰黑，区别于卡片 */
  --background: #09090b;
  --foreground: #fafafa;

  /* 第2层 */
  --muted: #18181b;
  --muted-foreground: #71717a;
  --secondary: #27272a;
  --secondary-foreground: #a1a1aa;

  /* 第3层：卡片 — 比背景亮 */
  --card: #18181b;
  --card-foreground: #fafafa;
  --popover: #1f1f23;
  --popover-foreground: #fafafa;

  /* 第4层 */
  --primary: #3b82f6;
  --primary-foreground: #ffffff;
  --accent: #27272a;
  --accent-foreground: #fafafa;

  /* 边框 — 更深，不突兀 */
  --border: #27272a;
  --input: #27272a;
  --ring: #3b82f6;

  /* 功能色 */
  --destructive: #dc2626;
  --destructive-foreground: #ffffff;
}
```

### 3.4 交互反馈色

按钮/链接的 hover 状态使用色调调整而非亮度调整：

```css
/* 正确：色相不变，透明度/亮度微调 */
.btn:hover {
  background: oklch(0.45 0.25 255 / 0.1); /* 10% 透明的主色叠加 */
}

/* 错误：直接变亮/变暗导致色相偏移 */
.btn:hover {
  background: #1d4ed8; /* 偏移的蓝色，不自然 */
}
```

## 四、字体与排版

### 4.1 字体使用

**当前问题：** 全文使用阿里巴巴普惠体，无层级区分。

**修复后层级：**

| 场景 | 字体 | 字重 | 说明 |
|------|------|------|------|
| 页面标题 H1 | Alibaba PuHuiTi 3.0 | 700 (Bold) | 24px |
| 区块标题 H2 | Alibaba PuHuiTi 3.0 | 600 (Semibold) | 16px |
| 卡片标题 H3 | Alibaba PuHuiTi 3.0 | 500 (Medium) | 15px |
| 正文内容 | Alibaba PuHuiTi 3.0 | 400 (Regular) | 14px |
| 次要说明 | Alibaba PuHuiTi 3.0 | 400 (Regular) | 13px，--muted-foreground |
| 代码/Markdown | JetBrains Mono / Fira Code | 400 | 14px，行高 1.6 |

### 4.2 行高规范

| 场景 | 行高 | 说明 |
|------|------|------|
| 标题 | 1.2 ~ 1.3 | 紧凑，视觉重量 |
| 正文段落 | 1.6 | 舒适阅读 |
| Markdown 内容 | 1.75 | 宽松，便于 Markdown 阅读 |
| 代码块 | 1.6 | 紧凑但可读 |

### 4.3 字间距

标题默认不使用负字间距，避免中文和日文界面在不同系统渲染时出现拥挤：

```css
h1, h2, h3 {
  letter-spacing: 0;
}
```

## 五、侧边栏（AppLayout → Sidebar）

### 5.0 应用内标题栏

Windows 主窗口关闭 Tauri 原生标题栏，只保留应用内自定义标题栏，避免出现两层头部。

**结构要求：**
- 高度 48px
- 背景使用 `--background`，与当前主题保持一致
- 底部使用 `1px --border` 分割
- 左侧显示应用状态和标题
- 右侧显示最小化、最大化、关闭按钮
- 整体为拖拽区域，按钮区使用 `no-drag`

```css
.titlebar {
  height: 48px;
  background: var(--background);
  border-bottom: 1px solid var(--border);
  color: var(--muted-foreground);
  -webkit-app-region: drag;
}

.titlebar-actions {
  -webkit-app-region: no-drag;
}
```

**Tauri 配置要求：**

```json
{
  "decorations": false
}
```

快速 Memo 小窗口可以继续保留原生标题栏，保持轻量窗口的系统移动和关闭体验。

### 5.1 当前问题

- 无阴影，无背景色区分
- Logo 区域与导航区无明显分割
- 折叠后只有 UC 两个字母，非常简陋
- 图标和文字间距不够讲究

### 5.2 视觉规范

**展开态（w-[220px]）：**

```
┌────────────────────────┐
│  [Logo]  UniComm      │  ← 高度 56px，底部 1px border
│───────────────────────│
│  快捷入口              │
│  📝  备忘录        12  │  ← 导航项 40px 高，8px 圆角
│───────────────────────│
│  状态                  │
│  [全部][普通][待办][完成]│
│───────────────────────│
│  设置                  │
│  ⚙️  设置              │
│                       │
│  [主题切换/就绪状态]    │
└────────────────────────┘
     背景: --card
     右侧: 1px --border
     悬浮: --shadow-sm
```

**关键样式：**

```css
.sidebar {
  background: var(--card);           /* 不是 background，是 card */
  border-right: 1px solid var(--border);
  box-shadow: var(--shadow-sm);    /* 微妙的浮起感 */
}

/* 导航项 hover */
.nav-item {
  border-radius: var(--radius-md); /* 8px 圆角 */
  transition: all 150ms ease;      /* 快速但不突兀的动画 */
}

.nav-item:hover {
  background: var(--accent);
}

/* 激活态 */
.nav-item.active {
  background: var(--accent);
  color: var(--accent-foreground);
  font-weight: 500;
}
```

### 5.3 折叠态（w-16）

折叠后显示图标，tooltip 显示文字：

```css
.sidebar.collapsed .nav-item {
  justify-content: center;
  width: 40px;
  height: 40px;
  margin: 0 auto;
}
```

### 5.4 过渡动画

```css
.sidebar {
  transition: width 200ms cubic-bezier(0.4, 0, 0.2, 1);
}
```

## 六、Memo 工作区（MemoWorkspace）

### 6.1 当前问题

- 三栏无间距，贴在一起
- 列表项 hover 效果过于简单（只有背景色变化）
- 编辑器区域像 textarea，无编辑氛围
- 顶部工具栏按钮无分组
- 空状态只有文字
- 原型中的通知中心和智能助手尚未实现，当前不在主导航展示

### 6.2 三栏布局规范

```
┌──────────────┬──────────────────┬────────────────────────────────────────┐
│ 应用侧边栏    │    Memo 列表区    │       编辑器                           │
│ 220px        │    320px         │       minmax(0,1fr)                    │
│              │ 搜索框            │  标题输入框                            │
│ 快捷入口      │ 新建 Memo         │  分组选择 / 状态选择 / 更新时间          │
│ 状态筛选      │ 分组筛选          │  工具栏 + 可视化/MD切换                 │
│ 设置          │ MemoItem         │  双栏编辑器 / 可视化 / MD 源码            │
│ 主题切换      │ MemoItem         │  底部：保存状态 / 归档 / 收藏 / 保存      │
└──────────────┴──────────────────┴────────────────────────────────────────┘
       ↑ 右边框                    ↑ 右边框                  ↑ 无边框
```

**间距：**
- 栏与栏之间：`1px` 边框分割，无额外间距（边框即分割）
- 左侧栏内：`space-sm` 水平内边距，`space-xs` 垂直间距
- 中间栏列表项之间：`1px` 底部边框（不是间距）

### 6.3 列表项（MemoItem）规范

**正常态：**

```css
.memo-item {
  padding: var(--space-sm) var(--space-md);
  border-bottom: 1px solid var(--border);
  cursor: pointer;
  transition: background 100ms ease;
}

.memo-item:hover {
  background: var(--accent);
}
```

**选中态：**

```css
.memo-item.selected {
  background: var(--accent);
  border-left: 2px solid var(--primary); /* 左侧主色竖线 */
}
```

**置顶/收藏标识：**
- 置顶：标题左侧显示 📌 图标，12px，主色
- 收藏：星标图标填充主色

### 6.4 编辑器区域规范

**编辑器功能范围：**

| 功能 | 当前行为 |
|------|----------|
| 可视化编辑 | 使用 Milkdown Crepe，编辑效果与阅读效果一致 |
| 双栏编辑 | 默认左侧为可视化编辑区，右侧为 MD 源码区，参考 Milkdown Playground 的编辑体验 |
| 常用排版 | 由 Crepe 内置 TopBar、浮动工具栏、块级菜单提供标题、加粗、斜体、列表、引用、链接、图片、表格、代码块、公式等能力 |
| 图片 | 选择本地图片，当前转 base64 后插入 Markdown 图片 |
| MD 源码 | 高级模式，直接编辑 Markdown 源码 |

**排版要求：**
- 标题输入框、分组/状态/更新时间位于编辑器顶部
- 分组选择器保持下拉形式，但使用紧凑尺寸：高度 28px、字体 12px、宽度约 120-140px，并带分组图标/标签
- 状态选择器使用三段式按钮展示正常、待办、完成，按钮内使用颜色圆点标识状态，避免高频操作隐藏在下拉框内
- 可视化/MD 切换与分组、状态、更新时间保持同一行，右侧对齐，不单独占一行
- 默认模式为双栏编辑，宽屏下左右并排展示；低分辨率下允许上下分区，且可切换到单独可视化或单独 MD 源码
- 标题为空时显示"请输入标题"占位符，不自动填充"无标题"等默认值
- 右侧工作区使用纵向弹性布局：顶部标题区和底部操作栏固定，中间编辑区 `min-height: 0; flex: 1; overflow: hidden`
- 长 Memo 只允许编辑器框内部滚动，页面主体和底部操作栏不能被内容挤出视口
- 可视化编辑器外框负责 `overflow-y: auto`，Milkdown/ProseMirror 内部不能使用 `100vh` 计算高度
- 主窗口默认宽度为 1440px，最小宽度为 1024px；低分辨率下双栏编辑区改为上下分区，保证按钮和正文不竖排挤压

**标题输入框：**

```css
.memo-title-input {
  border: none;                      /* 无边框 */
  background: transparent;           /* 透明背景 */
  font-size: 20px;
```

```css
  padding: var(--space-md);
  font-family: JetBrains Mono, monospace;
  font-size: 14px;

  color: var(--foreground);
  outline: none;

.memo-content-editor:focus {
```

**工具栏按钮：**

```css

  width: 32px;
  height: 32px;
```

## 七、设置页面（SettingsPanel）

- 整个页面是一个 `max-w-3xl mx-auto`，两侧无装饰

```css
.settings-card-header h2 {
```

## 十一、深色模式质量标准

### 11.1 深色模式原则

1. **不是简单的颜色反转** — 明暗调整要保持品牌感
2. **背景不是纯黑** — `#09090b` 优于 `#000000`
3. **文字不是纯白** — `#fafafa` 优于 `#ffffff`
4. **边框要淡** — 深色模式下边框 `rgba(255,255,255,0.1)`

### 11.2 深色模式颜色核验清单

- [ ] 背景与卡片有可辨别差异（不能融为一体）
- [ ] 输入框边框在深色下可见
- [ ] hover 状态在深色下有反馈
- [ ] focus ring 在深色下清晰可见
- [ ] 次要文字颜色足够淡，不干扰主要文字
- [ ] 无过亮的强调色造成眼睛疲劳
